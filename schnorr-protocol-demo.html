Schnorr Protocol (simplified)

João wants to prove that he knows the secret x such that g^x mod p = y, without revealing x. This is the core idea behind many modern digital signature systems.

🔒 João: knows x (the secret)  |  👁️ Thomas: only knows g, p, and y = g^x mod p
  • Setup: generate public parameters (g, p) and the public key y = g^x mod p
  • João chooses a random nonce r and computes R = g^r mod p
  • Thomas sends a random challenge c
  • João responds with s = r + c·x (without revealing x or r individually)
  • Thomas checks: g^s mod p == R · y^c mod p

Parameters

—
g (generator), p (prime)

Secret x (only João)

—
It is never transmitted

Public key y

—
y = g^x mod p

Nonce r

—
Random in each round

Commitment R

—
R = g^r mod p

Challenge c

—
Sent by Thomas

Response s

—
s = r + c·x

Verification

—
g^s mod p vs R·y^c mod p
