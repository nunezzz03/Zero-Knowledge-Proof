# Zero-Knowledge Proof Demos

This repository contains interactive web-based demonstrations for understanding Zero-Knowledge Proofs (ZKPs) as part of a Discrete Mathematics project.

## Files Included

### 1. Ali Baba Cave Demo (`ali-baba-cave-demo.html`)
This file provides an interactive visual demonstration of the classic Ali Baba Cave thought experiment, which serves as an intuitive introduction to zero-knowledge proofs.
* **Objective:** The Prover (João) tries to convince the Verifier (Thomas) that he knows the secret password to open a magical door inside a cave, without ever revealing the password itself.
* **Features:**
  * Visual representation of the cave paths (Path A and Path B).
  * Toggle mode to simulate whether João actually knows the password or is just cheating.
  * Step-by-step round execution showing how the Verifier's confidence increases with each successful round.
  * Live statistics tracking the number of rounds, successes, failures, and the statistical probability of undetected cheating.

### 2. Schnorr Protocol Demo (`schnorr-protocol-demo.html`)
This file breaks down the mathematical implementation of a Zero-Knowledge Proof using the Schnorr identification protocol.
* **Objective:** João proves he knows a secret value `x` (where `g^x mod p = y`) to Thomas, without ever transmitting or revealing `x`.
* **Features:**
  * Step-by-step interactive execution of the cryptographic protocol.
  * Transparent view of the variables at each stage:
    * **Setup:** Generation of public parameters and keys.
    * **Commitment (`R`):** Generated by João using a random nonce `r`.
    * **Challenge (`c`):** A random value requested by Thomas.
    * **Response (`s`):** Calculated by João using his secret `x`, nonce `r`, and challenge `c`.
    * **Verification:** Real-time mathematical verification (`g^s mod p == R · y^c mod p`) demonstrating why the protocol is mathematically sound.

## How to Use
Simply open the HTML files (`ali-baba-cave-demo.html` or `schnorr-protocol-demo.html`) directly in any modern web browser to interact with the simulations. No local server, backend, or additional dependencies are required. All interactive logic and styling are self-contained using standard JavaScript, HTML, and CSS.
